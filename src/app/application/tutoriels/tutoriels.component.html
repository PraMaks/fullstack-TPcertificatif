4) Pourquoi une SPA est plus performante?

Une SPA charge sa page sur le navigateur Web du client et sur le serveur. Cependant, elle se charge qu'une seule fois, puisque c'est le contenu de la page qui se change et non la page ce qui évite de charger une nouvelle page.

Quelques notions importantes sur Angular

1) Le fichier angular.json: C'est le fichier de configuration dans lequel qu'il est possible d'importer des styles, des assets et des configurations.

2) Le fichier package.json: C'est un fichier qui contient des packages de l'applications. En d'autres mots, il est possible de changer ce que les commandes npm font dans la partie scripts et de changer la version des dependances.

3) Selectors: C'est le nom qu'un composant Angular porte. Cela permet de plus facilement aller le chercher lors de al programmation.

4) Composant: C'est une partie de l'application qui affiche/contrôle une partie de la page. Ce dernier se fait genérer avec un fichier .html, .css et deux fichiers .ts.

5) Interpollation et "Banane": Deux paires d'accolades et une paire de parenthèses carrés avec une paire d'accolades(dans les parenthèses carrées)

Quelques notions importantes sur TypeScript(TS)

1) TS est un sur-ensemble de JavaScript (le code JavaScript correct peut être utilisé avec TypeScript)

2) TS est un language compilé et lorsque c'est fait, il génère des fichiers JS

3) La synthaxe du TS ressemble beaucoup à celle de Java.

4) Le code TS est plus facile à lire et propre que JS.

Mini Tutoriel sur les relations

C'est quoi des relations en informatique?

En programmation Orientée-Objet, il est nécessaire pour que les objets puissent se communiquer pour faire fonctionner les applications. Généralement, elles sont répresentées avec des diagrammes UML comme celui en dessous...

Il existe plusieurs relations différentes en Programmation Orientée-Objet

1) Association: Relation entre 2 objets qui ont un lien. Elle peut être one to one, one to many ou many to many.

2) Héritage: Relation parent-enfant entre 2 objets dans laquelle l'enfant hérite des caractéristiques du parent.

3) Aggregation: Relation entre 2 objets dans laquelle l'un est un attribut de l'autre et il n'y a pas besoin de l'instancier dans tout constructeur de l'objet qui l'a comme un attribut.

4) Composition: Relation entre 2 objets dans laquelle l'un est un attribut de l'autre, mais qui doit être instancié dans tout constructeur de l'objet qui l'a comme un attribut.

5) Implémentation: Relation entre 2 objets dans laquelle l'un des objets est une interface de l'autre.
